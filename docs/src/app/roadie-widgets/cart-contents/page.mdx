# CartContents

The day-grouped event and ticket list that powers the cart drawer's body —
exported on its own so you can render just the list on a full cart page.

- Groups events by day with sticky day headers
- Reserved seats render as range badges via the shared `formatSeatRange`
- Per-event remove with an inline confirm, and removals animate out
- Presentational: you pass it a `CartDetails`; it renders — no overlay, no fetching

It's the same component `CartDrawer` renders internally, so a `/cart` page and
the docked drawer stay visually identical over one shared core.

## Live demo

The way a `/cart` page would use it — in a normal page column, no overlay. The
frame below is a fixed-height "page area" so you can see `fillHeight` work. Use
the **React / Vue** tabs to compare the skins, and the toggles to test:

- **`fillHeight`** — on, the cart fills the frame: the footer pins to the bottom
  for a short cart and the empty state centres. Off, the cart sits at its natural
  height, top-aligned.
- **`hideFooter`** — drop the Total / Checkout footer.
- **Add event / Empty cart / Reset** — change the cart length, or empty it to see
  the last event animate out and the empty state animate in. Per-event removal
  (the trash icon) animates too. Runs on a canned cart.

## Installation

```bash
pnpm add @oztix/roadie-widgets
```

CartContents is presentational — lighter peers than the full drawer (no
react-query, no focus-trap):

```bash
pnpm add react react-dom @oztix/roadie-components motion @phosphor-icons/react
```

### Tailwind setup

CartContents' classes live in the package's compiled output, so Tailwind needs
to scan it — otherwise the styles never make it into your CSS. Add one import to
your global stylesheet, next to the core import:

```css
/* app/globals.css */
@import '@oztix/roadie-core/css';
@import '@oztix/roadie-widgets/css';
```

`@oztix/roadie-widgets/css` ships its own `@source` pointing at the package's
`dist`, so you never write a brittle `node_modules` path yourself. One line
covers every widget (cart contents and cart drawer).

## Import

```tsx
// React
import { CartContents } from '@oztix/roadie-widgets/cart-contents/react'
import type { CartDetails } from '@oztix/roadie-widgets/cart-contents/react'
```

```ts
// Vue
import { CartContents } from '@oztix/roadie-widgets/cart-contents/vue'
```

Either entry pulls in only the list — none of the drawer's overlay, drag,
focus-trap, or expiry code. The cart drawer composes this same component, so the
standalone page and the docked drawer stay identical.

## Usage

Pass a `CartDetails` and the presentational props; the host owns data fetching
and routing. Build a `CartDetails` however you like, or reuse the shared cart
client:

```tsx
import { createCartClient } from '@oztix/roadie-widgets/cart'
import { CartContents } from '@oztix/roadie-widgets/cart-contents/react'

const cart = createCartClient({ host: '' }) // same-origin
const details = await cart.getDetails(collectionId)

;<CartContents
  cart={details}
  onNavigate={(href) => router.push(href)}
  browseHref="/events"
  checkoutUrl={cart.checkoutUrl(details)}
  locale="en-AU"
  currency="AUD"
/>
```

- **`fillHeight`** — for a full `/cart`-style page. The root becomes a
  fill-height column so it fills whatever height its container provides: the
  footer pins to the bottom for short carts (no scroll overflow needed) and the
  empty state centres. Render it as just
  `<div className="min-h-full"><CartContents fillHeight … /></div>` — no external
  `flex-1`, `mt-auto`, or conditional `justify-center`. Leave it unset inside the
  drawer, which sizes its own panel. Composes with `stickyFooter`: `mt-auto`
  covers short carts, `stickyFooter` keeps the footer pinned while a long cart
  scrolls.
- **`hideFooter`** — drop the Total / fees / Checkout footer (e.g. when your
  page renders its own order summary).
- **`stickyFooter`** — pin the footer to the bottom of the nearest scrolling
  ancestor.
- **`onRemoveEvent` / `busy`** — wire per-event removal; `busy` disables the
  trash triggers while a remove is in flight.

When `cart` has no tickets, CartContents renders the empty state with the
`browseHref` call to action. Removing the last event animates the list out and
the empty state in (one shared presence tree), and under `fillHeight` the empty
state fills and centres in the column.
