# Mark

Highlighted text using the HTML mark element with intent variants.

## Import

```tsx
import { Mark } from '@oztix/roadie-components'
```

## Examples

### Default

The default intent is `info`.

```tsx-live

<Text>
  The Mark component is used to <Mark>highlight text</Mark> within content.
</Text>
```

### Intents

```tsx-live

<div className='flex flex-col gap-2'>
  <Text>
    This is <Mark intent='neutral'>neutral</Mark> highlighted text.
  </Text>
  <Text>
    This is <Mark intent='accent'>accent</Mark> highlighted text.
  </Text>
  <Text>
    This is <Mark intent='brand'>brand</Mark> highlighted text.
  </Text>
  <Text>
    This is <Mark intent='info'>info</Mark> highlighted text.
  </Text>
  <Text>
    This is <Mark intent='success'>success</Mark> highlighted text.
  </Text>
  <Text>
    This is <Mark intent='warning'>warning</Mark> highlighted text.
  </Text>
  <Text>
    This is <Mark intent='danger'>danger</Mark> highlighted text.
  </Text>
</div>
```

### With headings

```tsx-live

<div className='flex flex-col gap-2 text-center'>
  <Heading as='h1' className='text-display-ui-3'>
    Great events need great <Mark intent='accent'>partners</Mark>
  </Heading>
  <Heading as='h2' className='text-display-ui-5' emphasis='subtle'>
    Not just <Mark intent='warning'>great</Mark> ticketing software. Why?
  </Heading>
</div>
```
