# Image

A size-aware `<img>` that routes Oztix-CDN assets through the resize/transcode proxy.

Pass a `width` and the component requests a right-sized WebP from the Oztix CDN plus a 1x/2x `srcSet` ÔÇö cutting download bytes and decoded-bitmap memory. Non-Oztix URLs (and calls without a `width`) pass through as a plain `<img>`, so external thumbnails still work untouched.

Every image shows a subtle `bg-subtle` tint as a placeholder from first paint ÔÇö before the blur preview or full image arrives ÔÇö then drops it once the image loads, so it never lingers behind a transparent image. Override the placeholder surface with another semantic background utility (e.g. `className='bg-sunken'`).

## Import

```tsx
import { Image } from '@oztix/roadie-components'
```

## Examples

### Default

Without a `width`, `Image` renders a plain `<img>` ÔÇö no wrapper, no URL rewrite. It serves the original file (here the full 2752├ù1536 6.1 MB PNG), so pass a `width` to optimise ÔÇö see the next example.

```tsx-live
<Image
  src='https://assets.oztix.com.au/image/1226ab55-3d53-47f0-ab4c-cddcf02bb001.png'
  alt='Event artwork'
  className='w-full rounded-xl'
/>
```

### Resizing Oztix assets

For images served from the Oztix CDN, pass `width` (the 1x render width in CSS pixels). The component rewrites `src` to `?width=&format=webp` and emits a `[width, width * 2]` `srcSet` with `sizes="${width}px"`:

```tsx
<Image
  src='https://assets.oztix.com.au/image/1226ab55-3d53-47f0-ab4c-cddcf02bb001.png'
  alt='Show artwork'
  width={360}
/>
// ÔåÆ
// src    = ÔÇª1226ab55ÔÇª.png?width=360&format=webp
// srcSet = ÔÇª?width=360&format=webp 360w, ÔÇª?width=720&format=webp 720w
// sizes  = 360px
```

`width` alone scales proportionally. Add `height` to crop to a fixed box ÔÇö it's sent to the proxy and scaled with each `srcSet` entry to keep the ratio:

```tsx
<Image src={url} alt='Show artwork' width={360} height={240} />
// ÔåÆ src = ÔÇª?width=360&height=240&format=webp
//   srcSet = ÔÇªwidth=360&height=240ÔÇª 360w, ÔÇªwidth=720&height=480ÔÇª 720w
```

### Trimming transparent space

Logos often ship with baked-in transparent padding. Set `autotrim` to crop it server-side (`autotrim=1`) so the artwork fills the box. Best paired with a proportional `width` (no `height`) so the trimmed result keeps its own ratio.

```tsx
<Image src={logoUrl} alt='Event logo' width={300} autotrim />
// ÔåÆ ÔÇª?width=300&format=webp&autotrim=1
```

### Quality and advanced commands

`quality` (1ÔÇô100) trades fidelity for bytes. For anything else the proxy supports ÔÇö resize mode, anchor, background colour ÔÇö use the `params` escape hatch; entries are merged verbatim into the query.

```tsx
<Image
  src='https://assets.oztix.com.au/image/1226ab55-3d53-47f0-ab4c-cddcf02bb001.png'
  alt='Cropped card art'
  width={700}
  height={350}
  quality={70}
  params={{ rmode: 'crop', bgcolor: 'fff' }}
/>
// ÔåÆ ÔÇª?width=700&height=350&format=webp&quality=70&rmode=crop&bgcolor=fff
```

Supported `params` keys (ImageSharp.Web): `rmode`, `ranchor`, `rxy`, `rsampler`, `compand`, `orient`, `bgcolor`, `autoorient`, `pngcolortype`.

### Reserving layout

Pass `width` and `height` to reserve space and avoid layout shift ÔÇö the component sets the `width`/`height` attributes and an `aspect-ratio` style.

```tsx-live
<Image
  src='https://assets.oztix.com.au/image/1226ab55-3d53-47f0-ab4c-cddcf02bb001.png'
  alt='Event artwork'
  width={800}
  height={450}
  className='w-full rounded-xl'
/>
```

The source is a 2752├ù1536 6.1 MB PNG; resized to WebP it ships ~280 KB.

### Hero imagery

For images that stretch across breakpoints, pass an explicit `widths` ladder and `sizes`, and mark the LCP image with `priority` (eager load + `fetchpriority="high"`).

```tsx
<Image
  src={heroUrl}
  alt='Festival hero'
  width={1200}
  height={480}
  widths={[768, 1200, 1600, 2400]}
  sizes='100vw'
  priority
/>
```

### Art direction (different crops per breakpoint)

`widths`/`sizes` is resolution switching ÔÇö the *same* image at different sizes. When you need a genuinely different URL or crop per breakpoint (a tall portrait on mobile, a wide banner on desktop), pass `sources`. The component renders a `<picture>` with a `<source>` per entry and keeps `src` as the fallback `<img>`. Each source inherits `format`/`quality`/`autotrim`/`params`/`width` unless it overrides them.

```tsx-live
<Image
  src='https://assets.oztix.com.au/image/1226ab55-3d53-47f0-ab4c-cddcf02bb001.png'
  alt='Event artwork'
  width={1200}
  height={500}
  sizes='100vw'
  className='w-full rounded-xl'
  sources={[
    {
      media: '(max-width: 640px)',
      width: 640,
      height: 800,
      params: { rmode: 'crop' }
    }
  ]}
/>
```

Here desktop gets a wide 1200├ù500 crop and phones a tall 640├ù800 crop of the same asset ÔÇö pass a different `src` per source to serve entirely separate images.

### Blur-up placeholder

Set `placeholder='blur'` to show a tiny low-quality preview that fades into the full image on load ÔÇö the default loading treatment for content imagery. The preview is derived from the Oztix proxy automatically; pass `width`+`height` so layout is reserved while it loads.

```tsx-live
<Image
  src='https://assets.oztix.com.au/image/1226ab55-3d53-47f0-ab4c-cddcf02bb001.png'
  alt='Event artwork'
  width={800}
  height={450}
  placeholder='blur'
  className='w-full rounded-xl'
/>
```

For non-Oztix images (or to override the auto preview), pass `blurDataURL` ÔÇö a data URI or any small image URL.

### Deferred loading in carousels

Embla mounts every slide in the DOM, so native `loading='lazy'` over-fetches off-screen slides. Set `defer` to hold the `src` until the tile nears the viewport (IntersectionObserver). Layout is still reserved from `width`/`height`.

```tsx
<Carousel>
  <Carousel.Content>
    {events.map((event) => (
      <Carousel.Item key={event.id}>
        <Card href={event.href}>
          <Card.Image src={event.image} alt={event.title} width={360} defer />
        </Card>
      </Carousel.Item>
    ))}
  </Carousel.Content>
</Carousel>
```

## Helpers

For custom compositions (LQIP banners, CSS `background-image`, server-rendered `<img srcset>`), the pure URL helpers ship from `@oztix/roadie-core/image` without the React component:

```tsx
import {
  isOztixImageUrl,
  oztixImageAtWidth,
  oztixSrcSet
} from '@oztix/roadie-core/image'

oztixImageAtWidth(src, 600) // ÔÇª?width=600&format=webp
oztixImageAtWidth(src, 600, { autotrim: true }) // ÔÇª&autotrim=1
oztixSrcSet(src, [600, 1200]) // ÔÇª600w, ÔÇª1200w  (undefined for non-Oztix URLs)
isOztixImageUrl(src) // true for assets.oztix.com.au + the CloudFront distros
```

### Hero image with blur (Vue)

Vue consumers don't get the React component, but the helpers give you the same result. Derive a tiny LQIP and a responsive `srcSet` from the source, then fade the full image in over the blurred preview. Style with Roadie/Tailwind utilities ÔÇö only the dynamic LQIP URL needs an inline `:style`.

```vue
<script setup lang="ts">
import { ref } from 'vue'
import { oztixImageAtWidth, oztixSrcSet } from '@oztix/roadie-core/image'

const props = defineProps<{ src: string; alt: string }>()

// Pure URL helpers ÔÇö no network, no React.
const lqip = oztixImageAtWidth(props.src, 24, { quality: 30 })
const full = oztixImageAtWidth(props.src, 1600)
const srcset = oztixSrcSet(props.src, [768, 1200, 1600, 2400])

const loaded = ref(false)
</script>

<template>
  <div class="relative aspect-[16/9] w-full overflow-hidden rounded-5xl">
    <!-- Blurred low-res preview, fades out once the full image is ready -->
    <div
      aria-hidden="true"
      class="absolute inset-0 scale-110 bg-cover bg-center blur-lg transition-opacity duration-500"
      :class="loaded ? 'opacity-0' : 'opacity-100'"
      :style="{ backgroundImage: `url(${lqip})` }"
    />
    <!-- Full image: eager + high priority because a hero is the LCP element -->
    <img
      :src="full"
      :srcset="srcset"
      sizes="100vw"
      :alt="alt"
      width="1600"
      height="900"
      fetchpriority="high"
      decoding="async"
      class="absolute inset-0 size-full object-cover transition-opacity duration-500"
      :class="loaded ? 'opacity-100' : 'opacity-0'"
      @load="loaded = true"
    />
  </div>
</template>
```

A hero is above the fold, so it loads eagerly (no `loading='lazy'`) with `fetchpriority='high'`. For a cached image the `@load` may have already fired before hydration ÔÇö guard with `if ((e.target as HTMLImageElement).complete) loaded = true` on mount if that matters for your app.

## Guidelines

- Pass `width` for any Oztix-CDN image ÔÇö it is the switch that turns on resizing.
- Pass `height` too whenever you know the aspect ratio, to prevent layout shift.
- Mark the single largest above-the-fold image `priority`; leave everything else lazy.
- Use `defer` for images inside horizontally-scrolled surfaces like `Carousel`.
- Don't reach for `width` on a non-Oztix URL expecting a resize ÔÇö those pass through untouched.

## Accessibility

- `alt` is required. Use a concise description, or an empty string (`alt=''`) for purely decorative images paired with `aria-hidden` on a parent.
- `width`/`height` reserve layout, reducing cumulative layout shift for assistive-tech and keyboard users alike.
