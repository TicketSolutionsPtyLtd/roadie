# LinkIconButton

> **Deprecated.** Use [`<IconButton href={…}>`](/components/icon-button) and wire
> [`RoadieLinkProvider`](/foundations/linking) at the app root for
> client-side routing instead. `LinkButton` and `LinkIconButton` remain
> exported and continue to work for back-compat, but will be removed in
> v3.0.0.

An icon-only link styled as a button. Shares all visual styles with [IconButton](/components/icon-button).

## Import

```tsx
import { LinkIconButton } from '@oztix/roadie-components'
```

## Examples

### Default

Always provide an `aria-label` since there is no visible text.

```tsx-live
<LinkIconButton href='#' aria-label='Settings'>
  <Gear weight='bold' />
</LinkIconButton>
```

### With custom component

Use the `as` prop to render with a custom component like Next.js `Link`.

```tsx
import Link from 'next/link'

<LinkIconButton as={Link} href='/settings' aria-label='Settings'>
  <GearIcon weight='bold' />
</LinkIconButton>
```

## Guidelines

- Always provide an `aria-label` to describe the action since there is no visible text.
- For links with visible text, use [LinkButton](/components/link-button) instead.
- The `as` prop accepts any component that forwards `className` and `href` (e.g., Next.js `Link`, React Router `Link`).

## Accessibility

- **Keyboard**: `Enter` activates the link. Focus is indicated by a semi-transparent ring via `is-interactive`.
- **ARIA**: Renders as a native `<a>` element by default. Screen readers announce it as a link, not a button.
- Requires an `aria-label` prop (enforced by TypeScript) since there is no visible text.
