# Typography

Typography in Roadie v2 is built on Tailwind CSS v4 with fluid `clamp()`-based font sizes and pre-composed text style utilities.

## Font families

We use two font families, registered via `@theme`:

- **Intermission** (`font-sans`) â€” Oztix's customised version of Inter with OpenType features for legibility. Variable weight (100â€“900).
- **IBM Plex Mono** (`font-mono`) â€” Monospace font for code elements.

Both are loaded via CDN with `font-display: swap`.

### Intermission features

Intermission has these OpenType features permanently enabled:

- **case**: Case-sensitive forms
- **ss03**: Round quotes & commas
- **cv01**: Alternate one
- **cv02**: Open four
- **cv03/04**: Open six and nine
- **cv05**: Lower case L with tail
- **cv08**: Upper-case I with serif
- **cv09**: Flat top three
- **cv10**: Capital G with spur
- **cv11**: Single-storey a

## Usage patterns

### Body text

Use semantic HTML elements with utility classes. No wrapper component needed â€” the body reset provides `text-ui` defaults.

```tsx
<p>Default body text â€” inherits body styles</p>
<p className="text-lg">Large text</p>
<p className="text-subtle">Secondary text</p>
<p className="text-strong">Emphasised text</p>
<p className="text-subtler text-sm">Meta / hint text</p>
<span className="text-sm">Inline small text</span>
```

### Headings

Use raw heading elements with display style utilities. Always pair a `text-display-*` class with a text color class like `text-strong`.

```tsx
{/* App UI headings */}
<h1 className="text-display-ui-1 text-strong">Dashboard</h1>
<h2 className="text-display-ui-3 text-strong">Section</h2>
<h3 className="text-display-ui-5 text-strong">Subsection</h3>

{/* Long-form content headings */}
<h1 className="text-display-prose-1 text-strong">Article title</h1>
<h2 className="text-display-prose-3 text-strong">Chapter</h2>
```

### Inline code

```tsx
import { Code } from '@oztix/roadie-components'

<Code>npm install @oztix/roadie-components</Code>
<Code intent="danger">error: something broke</Code>
```

### Prose (rich content)

The `Prose` component styles nested HTML for CMS output, markdown, or user-generated content. It sets appropriate typography, spacing, and element styles.

```tsx
import { Prose } from '@oztix/roadie-components'

<Prose>
  <h2>Article heading</h2>
  <p>Body paragraph with <strong>bold</strong> and <a href="#">links</a>.</p>
  <ul>
    <li>List item one</li>
    <li>List item two</li>
  </ul>
  <blockquote>A pull quote</blockquote>
</Prose>
```

## Semantic elements guide

| Element | When to use | Example |
|---------|-------------|---------|
| `<p>` | Body text, descriptions | `<p className="text-subtle">Description</p>` |
| `<span>` | Inline text within other elements | `<span className="text-sm">Label</span>` |
| `<h1>`â€“`<h6>` | Section headings (maintain hierarchy) | `<h2 className="text-display-ui-3 text-strong">` |
| `<strong>` | Important text | `<strong>Required</strong>` |
| `<em>` | Emphasis / stress | `<em>Note:</em>` |
| `<a>` | Links | Style with `underline underline-offset-2` |
| `<Code>` | Inline code, tokens, commands | `<Code>bg-default</Code>` |

## Semantic line heights and letter spacing

| Token | Line height | Letter spacing |
|-------|------------|----------------|
| `leading-display` / `tracking-display` | 1.2 | -0.02em |
| `leading-ui` / `tracking-ui` | 1.35 | -0.01em |
| `leading-prose` / `tracking-prose` | 1.5 | -0.01em |
| `leading-code` / `tracking-code` | 1.625 | 0em |

## Best practices

1. **Use semantic HTML elements** â€” `<p>` for body, `<h1>`-`<h6>` for headings, `Code` for code
2. **Use text style utilities** â€” `text-display-ui-3 text-strong` on a `<h2>`
3. **Let fluid type handle responsiveness** â€” no manual `md:text-lg` needed for font sizes
4. **Use emphasis classes** â€” `text-subtle` not manual color classes
5. **Maintain heading hierarchy** â€” h1 -> h2 -> h3, match semantic level to visual style
