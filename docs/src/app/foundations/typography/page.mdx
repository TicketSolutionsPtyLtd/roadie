# Typography

Typography in Roadie v2 is built on Tailwind CSS v4 with fluid `clamp()`-based font sizes and pre-composed text style utilities.

## Font families

We use two font families, registered via `@theme`:

- **Intermission** (`font-sans`) — Oztix's customised version of Inter with OpenType features for legibility. Variable weight (100–900).
- **IBM Plex Mono** (`font-mono`) — Monospace font for code elements.

Both are loaded via CDN with `font-display: swap`.

### Intermission features

Intermission has these OpenType features permanently enabled:

- **case**: Case-sensitive forms
- **ss03**: Round quotes & commas
- **cv01**: Alternate one
- **cv02**: Open four
- **cv03/04**: Open six and nine
- **cv05**: Lower case L with tail
- **cv08**: Upper-case I with serif
- **cv09**: Flat top three
- **cv10**: Capital G with spur
- **cv11**: Single-storey a

## Font size scale

Font sizes use `clamp()` for fluid scaling from `lg` upward:

| Token | Value |
|-------|-------|
| `text-xs` | 0.75rem (12px) |
| `text-sm` | 0.875rem (14px) |
| `text-base` | 1rem (16px) |
| `text-lg` | clamp(1.125rem → 1.25rem) |
| `text-xl` | clamp(1.25rem → 1.5rem) |
| `text-2xl` | clamp(1.5rem → 2rem) |
| `text-3xl` | clamp(1.75rem → 2.5rem) |
| `text-4xl` | clamp(2rem → 3rem) |
| `text-5xl` | clamp(2.25rem → 4rem) |
| `text-6xl` | clamp(2.5rem → 4.75rem) |
| `text-7xl` | clamp(2.75rem → 6rem) |

## Text style utilities

Pre-composed text styles combine font size, weight, line height, and letter spacing:

### Display styles (headings)

**UI context** (compact, for app interfaces):

| Utility | Size | Weight |
|---------|------|--------|
| `text-display-ui-1` | 4xl | bold |
| `text-display-ui-2` | 3xl | bold |
| `text-display-ui-3` | 2xl | bold |
| `text-display-ui-4` | xl | bold |
| `text-display-ui-5` | lg | semibold |
| `text-display-ui-6` | base | semibold |

**Prose context** (larger, for long-form content):

| Utility | Size | Weight |
|---------|------|--------|
| `text-display-prose-1` | 5xl | black (900) |
| `text-display-prose-2` | 4xl | extrabold |
| `text-display-prose-3` | 3xl | bold |
| `text-display-prose-4` | 2xl | bold |
| `text-display-prose-5` | xl | bold |
| `text-display-prose-6` | lg | bold |

### Body text styles

| Utility | Size | Line height | Use case |
|---------|------|-------------|----------|
| `text-ui` | base | 1.35 | App interface text |
| `text-ui-meta` | sm | 1.35 | Secondary UI text |
| `text-prose` | base | 1.5 | Long-form content |
| `text-code` | sm | 1.625 | Code blocks |

## Components

### Text

```tsx
import { Text } from '@oztix/roadie-components'

<Text>Default body text</Text>
<Text size="lg">Large text</Text>
<Text emphasis="subtle">Subtle secondary text</Text>
<Text emphasis="strong">Bold emphasis text</Text>
<Text intent="accent">Accent-colored text</Text>
<Text as="span" size="sm">Inline small text</Text>
```

### Heading

```tsx
import { Heading } from '@oztix/roadie-components'

<Heading as="h1" className="text-display-prose-1">Page title</Heading>
<Heading as="h2" className="text-display-ui-3">Section heading</Heading>
<Heading as="h3" className="text-display-ui-4">Subsection</Heading>
<Heading emphasis="subtle" className="text-display-ui-5">Subtitle</Heading>
```

Or use the text style utilities directly on any element:

```tsx
<h1 className="text-display-prose-1">Page title</h1>
<h2 className="text-display-ui-3">UI section</h2>
<p className="text-prose">Body paragraph</p>
```

### Code

```tsx
import { Code } from '@oztix/roadie-components'

<Code>npm install @oztix/roadie-components</Code>
<Code intent="danger">error: something broke</Code>
```

## Semantic line heights and letter spacing

| Token | Line height | Letter spacing |
|-------|------------|----------------|
| `leading-display` / `tracking-display` | 1.2 | -0.02em |
| `leading-ui` / `tracking-ui` | 1.35 | -0.01em |
| `leading-prose` / `tracking-prose` | 1.5 | -0.01em |
| `leading-code` / `tracking-code` | 1.625 | 0em |

## Best practices

1. **Use components for semantic meaning** — `Text` for body, `Heading` for headings, `Code` for code
2. **Use text style utilities for one-off styling** — `text-display-ui-3` on a raw `<h2>`
3. **Let fluid type handle responsiveness** — no manual `md:text-lg` needed for font sizes
4. **Use emphasis props** — `emphasis="subtle"` not manual color classes
5. **Maintain heading hierarchy** — h1 → h2 → h3, use `level` prop on Heading
